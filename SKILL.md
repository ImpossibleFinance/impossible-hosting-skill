# ifhost — Deploy to Impossible Hosting

Deploy any Dockerized app to the cloud with one command. Each app gets its own isolated VM, HTTPS URL, object storage, and auto-scaling.

## Install

```bash
curl -fsSL https://impossible-api.fly.dev/install | sh
```

## Workflow

```bash
# 1. Login (one-time)
ifhost login

# 2. Configure (agent decides specs based on project)
ifhost init --app <name> --port <port> --memory <mb> [flags]

# 3. Deploy
ifhost deploy
```

## ifhost init flags

| Flag | Default | Description |
|------|---------|-------------|
| `--app` | (required) | App name → becomes `<name>.fly.dev` |
| `--port` | from Dockerfile | Port the app listens on |
| `--memory` | 256 | RAM in MB (256, 512, 1024, 2048, 4096) |
| `--cpus` | 1 | CPU count (1, 2, 4, 8) |
| `--cpu-kind` | shared | `shared` or `performance` |
| `--cmd` | Dockerfile CMD | Startup command override |
| `--autostop` | true | Set `false` for apps that take >60s to boot |
| `--min-machines` | 0 | Set `1` for no cold starts |
| `--storage` | cloud | `cloud` (S3 bucket) or `local` (volume at /data) |

## ifhost deploy flags

| Flag | Description |
|------|-------------|
| `--env KEY=VALUE` | Set env var (repeatable) |
| `--secret KEY=VALUE` | Set secret (repeatable, not logged) |
| `--cmd "..."` | Override startup command |
| `--port N` | Override container port |
| `--yes` | Skip confirmation prompts |
| `--json` | Output as JSON |

## Common patterns

### Static site (nginx)
```bash
ifhost init --app my-site --port 80 --memory 256
ifhost deploy
```

### Node.js API
```bash
ifhost init --app my-api --port 3000 --memory 512
ifhost deploy --env DATABASE_URL=postgres://...
```

### Heavy app (AI agent, ML model)
```bash
ifhost init --app my-agent --port 3000 --memory 1024 --cpus 2 --autostop=false --min-machines 1 --cmd "setup.sh && start.sh"
ifhost deploy --env OPENAI_API_KEY=sk-... --env TELEGRAM_BOT_TOKEN=...
```

## After deploy

```bash
ifhost describe --app <name>              # Full app context
ifhost machines logs --app <name>         # Stream logs
ifhost machines exec --app <name> -- env  # Run command inside container
ifhost machines stop --app <name>         # Stop (no cost while stopped)
ifhost machines start --app <name>        # Start again
ifhost machines destroy --app <name> --yes  # Delete everything
```

## Traps to avoid

1. **PORT**: Must match what the app listens on. Check Dockerfile EXPOSE.
2. **RAM**: Node.js needs 512MB+, ML/AI needs 1024MB+. Default 256 causes silent OOM.
3. **AUTOSTOP**: Heavy apps get killed before booting. Set `--autostop=false --min-machines 1`.
4. **ENV VARS**: API keys go as `--env`, NOT in config files (they get overwritten on restart).
5. **CMD**: Apps needing setup before start use `--cmd "migrate && serve"`.
6. **LOCK FILES**: If an app hangs on restart, prepend `rm -f /tmp/<app>*.lock &&` to --cmd.

## Requirements

- A `Dockerfile` in the project root
- `impossible.toml` (generated by `ifhost init`)
